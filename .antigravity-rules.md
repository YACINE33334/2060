# قواعد Antigravity — تصميم الواجهة فقط

أنت مسؤول عن **التصميم والديزاين** فقط. لا تغير المنطق، الـ Backend، أو بنية الـ EJS.

---

## مهمتك
- تحسين الشكل والـ CSS (الألوان، الخطوط، المسافات، الظلال)
- تحسين الـ UX (الأزرار، البطاقات، التنقل)
- إضافة أيقونات أو رسوم توضيحية
- جعل الواجهة responsive وجميلة

## ما لا تفعله أبداً
- **لا تغيّر** بنية الـ EJS
- **لا تضيف** حقول جديدة للنماذج بدون موافقة
- **لا تضيف** routes أو server logic
- **لا تحذف** أو تغيّر أسماء الـ `action` أو `name` في الـ forms

---

## قواعد EJS المطلقة (إلا تخرّبها!)

### 1. لا تضع `<%= %>` أو `<%- %>` داخل Template Literal

**❌ خطأ — يسبب crash:**
```javascript
const body = `
  <input value="<%= config.storeName %>">   <!-- ممنوع! -->
`;
```

**✅ صح:**
```javascript
const esc = (s) => (s || '').replace(/"/g, '&quot;').replace(/</g, '&lt;');
const body = `
  <input value="${esc(config.storeName)}">   <!-- استخدم esc() -->
`;
```

### 2. داخل `<% ... %>` فقط استخدم JavaScript عادي
- استخدم `${variable}` للقيم داخل الـ template literal
- استخدم `esc()` لأي نص يأتي من المستخدم أو الـ config

### 3. بنية الملف
```ejs
<%
  const esc = (s) => (s || '').replace(/"/g, '&quot;').replace(/</g, '&lt;');
  const body = `
    ... المحتوى HTML هنا ...
    ... استخدم ${esc(config.xxx)} للقيم ...
  `;
%>
<%- include('admin-layout', { body, config, currentPage: 'xxx', pageTitle: 'عنوان' }) %>
```

### 4. أسماء الـ forms والـ inputs — لا تغيّرها
- `name="store_name"` `name="currency"` `name="facebook_pixel"` إلخ
- `action="/admin/settings"` `action="/admin/products"` إلخ

---

## الملفات التي يمكنك تصميمها
- `views/*.ejs` — التصميم فقط، بدون تغيير بنية EJS أو أسماء الـ forms

## الملفات التي لا تلمسها
- `server.js`
- `db.js`
- `config.json`
- `package.json`
